**serde_json persistence to /cache/agent_state.json enables AI state dots to appear for all sessions (active/idle/waiting) across session switches, plugin reloads, and Zellij instances**

## Accomplishments
- Added serde_json dep; `AgentState` now serializes to stable JSON (`"active"`, `"idle"`, `"waiting"`, `"unknown"`)
- `save_ai_states()` writes `BTreeMap<String, AgentState>` to `/cache/agent_state.json` on every state change
- `load_ai_states()` restores state on plugin startup (load()) and on every Timer tick (~10s refresh)
- Passive cross-session detection: `extract_active_command` now called for ALL sessions, not just the current one
- Active command display (detail line) now works for non-current sessions — both `has_command` check and rendering pattern fixed
- Stale AI states pruned from `ai_states` map on every SessionUpdate in both discovery and non-discovery paths

## Decisions Made
- Used path-qualified `serde::Serialize, serde::Deserialize` in the derive macro rather than adding a `use serde::{...}` import — avoids any potential conflict with the `zellij_tile::prelude::*` wildcard import
- Timer returns `true` unconditionally after adding `load_ai_states()` — simpler than tracking whether states actually changed; 10s re-renders are negligible overhead
- Write path is pipe handlers only; timer is read-only — prevents multiple sidebar instances fighting over `/cache` writes
